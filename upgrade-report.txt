1. FRONTEND PERFORMANCE — CODE SPLITTING & ERROR HANDLING
================================================================================

[MODIFIED] frontend/src/App.js
  - Semua 15 halaman (page) kini menggunakan React.lazy() untuk code splitting.
  - Setiap page di-load on-demand (lazy) sehingga initial bundle size JAUH
    lebih kecil. User tidak perlu menunggu seluruh aplikasi ter-download
    untuk bisa menggunakan halaman pertama.
  - Ditambahkan <Suspense> wrapper dengan LoadingFallback premium.
  - Ditambahkan <ErrorBoundary> wrapper untuk mencegah blank white screen
    jika terjadi error di salah satu komponen React.
  - Pages yang di-lazy-load:
      MarketingLandingPage, LandingPage, LoginPage, ChatPage, AboutYouPage,
      AuthCallback, VerifyEmailPage, AcademicPage, AdminDashboard, AdminLogin,
      HelpCenterPage, TermsPoliciesPage, ReportBugPage, TentangPage, DocsPage

[NEW] frontend/src/components/common/ErrorBoundary.jsx
  - Komponen ErrorBoundary class-based (React best practice).
  - Menampilkan UI premium jika terjadi crash (ikon, pesan, tombol reload/home).
  - Menampilkan error details di mode development saja.

[NEW] frontend/src/components/common/LoadingFallback.jsx
  - Komponen loading premium untuk Suspense boundaries.
  - Animasi pulse dan bounce dots dengan branding Sapa Tazkia.
  - Ditampilkan saat page sedang di-download pertama kali.

5. BACKEND — PROCESS MANAGEMENT & DATABASE INDEXING
================================================================================

[MODIFIED] backend/ecosystem.config.js (PM2 Config)
  - Ditambahkan node_args: '--max-old-space-size=512' untuk optimalisasi memori.
  - ignore_watch: menghindari restart tidak perlu dari node_modules, logs, dll.
  - kill_timeout: 10 detik untuk graceful shutdown.
  - listen_timeout: 8 detik batas tunggu koneksi.
  - exp_backoff_restart_delay: exponential backoff saat restart berulang.
  - min_uptime: 10 detik minimum uptime sebelum dianggap stabil.
  - log_type: 'json' untuk structured logging yang mudah di-parse.
  - log_date_format: ditambahkan timezone.

[MODIFIED] backend/prisma/schema.prisma (Database Indexes)
  - User model:
      @@index([email])            — pencarian user by email
      @@index([status])           — filter user by status
      @@index([lastLogin])        — sorting by last login
      @@index([programStudiId])   — filter by program studi
  - Session model:
      @@index([userId])           — lookup session by user
      @@index([expiresAt])        — cleanup expired sessions
      @@index([isActive])         — filter active sessions
  - Conversation model:
      @@index([userId])           — lookup conversations by user
      @@index([updatedAt])        — sorting by recent activity
  - Message model:
      @@index([conversationId])   — lookup messages by conversation
      @@index([createdAt])        — sorting messages chronologically
  TOTAL: 11 index baru ditambahkan.

  CATATAN: Setelah deploy, jalankan:
    npx prisma db push
  untuk menerapkan index baru ke database production.
